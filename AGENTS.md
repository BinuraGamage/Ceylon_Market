## 6. Firebase Rules ÔÇö Important Constraints

When writing Firestore queries, respect these security boundaries:
- A seller can only **read/write products** where `shopId == their own shopId`
- A customer can only **read/write their own cart** (`/users/{uid}/cart`)
- A customer can only **write one review per product** (enforced in Security Rules)
- Only `admin` role users can read/write `/ads`
- `customRequests` are readable by the `customerId`, assigned `shopId` owner, and assigned `designerId`
- Orders are readable by `customerId` and the `shopId` seller

## 8. UI & Theming Rules

- **Never hardcode colors** ÔÇö always use `AppColors.primary`, etc. from `app_colors.dart`
- **Never hardcode text styles** ÔÇö always use `AppTextStyles.heading1`, etc.

## 11. Code Quality Rules

These rules apply to every agent-generated file:

1. **No magic strings** ÔÇö all Firestore paths via `FirestorePaths`, all route names via named constants
2. **No `print()` statements** in production code ÔÇö use `debugPrint()` or a proper logger
3. **Every async function must have try/catch** ÔÇö never let unhandled Firestore/auth exceptions crash the app
4. **Null safety enforced** ÔÇö no `!` force-unwrap without a null check and comment explaining why it's safe
5. **Dispose resources** ÔÇö any `StreamSubscription` must be cancelled in `dispose()`
6. **No business logic in widgets** ÔÇö widgets call providers/services; providers contain logic
7. **One widget per file** ÔÇö unless the secondary widget is private and tiny (<30 lines)
8. **All models must implement `fromMap(Map)` and `toMap()`** ÔÇö consistent with Firestore doc format
9. **Every screen must handle loading, error, and empty states**
10. **Soft-delete only** ÔÇö never `.delete()` a Firestore doc; set `isActive: false` instead

```dart
// Ôťů Correct model pattern
class ProductModel extends Equatable {
  final String productId;
  final String name;
  final double price;
  // ...

  factory ProductModel.fromMap(Map<String, dynamic> map, String id) {
    return ProductModel(
      productId: id,
      name: map['name'] as String,
      price: (map['price'] as num).toDouble(),
      // ...
    );
  }

  Map<String, dynamic> toMap() => {
    'name': name,
    'price': price,
    // Note: never include productId in toMap() ÔÇö it's the doc ID
  };

  @override
  List<Object?> get props => [productId, name, price];
}
```
